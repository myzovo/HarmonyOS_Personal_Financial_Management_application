1项目概述

本项目是一个基于 HarmonyOS 平台开发的个人记账应用（记账本），采用 ArkTS 语言和 Stage 模型架构，运行于手机设备。应用提供了完整的账单录入、分类管理、统计分析和数据导出功能，帮助用户轻松管理日常收支。

2设计思路

2.1 技术选型

选择 HarmonyOS 原生开发框架，使用 ArkTS（基于 TypeScript 的声明式 UI 语言）配合 Stage 模型应用架构，充分利用系统提供的 UIAbility、RDB 数据库、Canvas 绘图等能力。

2.2 页面导航设计

采用底部 TabBar + 页面栈的混合导航模式。首页、账单列表、统计、导出四个主页面通过底部导航栏切换，记账页面通过 pushUrl 压入页面栈，支持返回操作。

2.3 数据模型设计

以 Bill 接口为核心数据模型，包含 id、金额、类型（支出/收入）、分类、备注、日期等字段。使用 RDB 关系型数据库持久化存储，通过 DatabaseHelper 单例类封装所有数据库操作。

2.4 UI 设计原则

遵循简洁直观的设计风格，统一使用 ThemeColors 集中管理配色，支持深色模式。记账页面采用自定义数字键盘设计，提升录入效率和用户体验。

4功能实现

4.1 首页仪表盘

首页展示今日和本月的收支汇总卡片（总支出、总收入、结余），以及最近 5 条账单记录。通过 DatabaseHelper 的 getTodaySummary() 和 getMonthSummary() 方法实时查询统计数据。底部 TabBar 支持在首页、账单、统计、导出之间切换。

4.2 记账功能

记账页面采用自定义 UI 设计，包含以下核心交互：

自定义数字键盘

支持按月份左右切换和按周筛选两种查看模式。账单按日期分组展示，每条记录显示分类图标、金额、类型和备注。点击可编辑，长按可删除。

5核心代码

5.1 数据库初始化（DatabaseHelper）

采用单例模式封装 RDB 数据库操作，提供账单的完整 CRUD 和统计功能：

utils/DatabaseHelper.ets ArkTS 
export class DatabaseHelper {
  private static instance: DatabaseHelper | null = null
  private rdbStore: relationalStore.RdbStore | null = null

  // 单例获取
  static getInstance(): DatabaseHelper {
    if (!DatabaseHelper.instance) {
      DatabaseHelper.instance = new DatabaseHelper()
    }
    return DatabaseHelper.instance
  }

  // 初始化数据库，创建 bills 表
  async initDatabase(context: common.UIAbilityContext): Promise<void> {
    const STORE_CONFIG: relationalStore.StoreConfig = {
      name: DB_NAME,
      securityLevel: relationalStore.SecurityLevel.S1
    }
    this.rdbStore = await relationalStore.getRdbStore(context, STORE_CONFIG)

    const sql = `CREATE TABLE IF NOT EXISTS bills (
      id TEXT PRIMARY KEY, type INTEGER NOT NULL,
      amount REAL NOT NULL, category TEXT NOT NULL,
      remark TEXT DEFAULT '', date TEXT NOT NULL,
      createTime TEXT NOT NULL, updateTime TEXT NOT NULL
    )`
    await this.rdbStore!.executeSql(sql)
  }
}

5.2 记账页面核心逻辑

记账页面使用 Stack 布局实现分层显示：主内容区 + 弹窗浮层。数字键盘通过 @Builder 方法构建，按键事件通过私有方法处理：

pages/AddBill.ets ArkTS 
@Entry
@Component
struct AddBill {
  @State amountStr: string = ''
  @State selectedCategory: string = ''
  @State typeTabIndex: number = 0

  build() {
    Stack({ alignContent: Alignment.TopStart }) {
      Column() {
        this.StatusBar()      // 模拟状态栏
        this.NavBar()        // 导航栏
        Scroll() {
          Column() {
            this.TypeTabs()     // 支出/收入/其他
            this.AmountSection() // 金额输入
            this.CategoryGrid()  // 分类图标
            this.InfoList()      // 时间/备注
          }
        }
        this.NumericKeyboard() // 底部数字键盘
      }
      this.RemarkDialog()    // 备注弹窗浮层
    }
  }

  // 按键处理逻辑
  private onNumKeyPress(label: string): void {
    if (label === '.') {
      if (!this.amountStr.includes('.')) {
        this.amountStr += this.amountStr.length === 0 ? '0.' : '.'
      }
    } else {
      if (this.amountStr.includes('.')) {
        const decimal = this.amountStr.split('.')[1]
        if (decimal && decimal.length >= 2) return
      }
      this.amountStr += label
    }
  }
}

5.3 Canvas 饼图绘制

饼图组件使用 Canvas API 手动绘制扇形、标签和图例，支持触摸交互和入场动画：

components/PieChart.ets ArkTS 
@Component
export struct PieChart {
  @Prop data: PieDataItem[] = []
  @Prop chartSize: number = 200
  @Prop title: string = ''
  @State currentIndex: number = -1

  private settings = new RenderingContextSettings(true)
  private context = new CanvasRenderingContext2D(this.settings)

  build() {
    Column() {
      Canvas(this.context)
        .width(this.chartSize)
        .height(this.chartSize)
        .onReady(() => { this.drawPie() })
        .onTouch((event) => { this.handleTouch(event) })
    }
  }

  private drawPie(): void {
    const total = this.data.reduce((sum, item) => sum + item.value, 0)
    let startAngle = -Math.PI / 2
    const cx = this.chartSize / 2
    const cy = this.chartSize / 2
    const radius = this.chartSize / 2 - 20

    for (const item of this.data) {
      const sweep = (item.value / total) * Math.PI * 2
      this.context.beginPath()
      this.context.moveTo(cx, cy)
      this.context.arc(cx, cy, radius, startAngle, startAngle + sweep)
      this.context.fillStyle = item.color
      this.context.fill()
      startAngle += sweep
    }
  }
}

6遇到的问题及解决方案

编译错误 Type 'void & Promise<RdbStore>' is not assignable to type 'RdbStore | null'
✅ 解决方案

getRdbStore() API 签名不需要第三个参数 DB_VERSION，移除后编译通过。同时对 rdbStore 使用非空断言（!）避免 "Object is possibly null" 错误。

编译错误 'relationalStore' has no exported member named 'ValueBucket'. Did you mean 'ValuesBucket'?
✅ 解决方案

API 类型名称拼写错误，将 ValueBucket 修正为 ValuesBucket（注意多一个 s）。

编译错误 Type 'number | undefined' is not assignable to type 'number'（DatePicker）
✅ 解决方案

DatePickerResult 的 year、month、day 属性可能为 undefined，使用空值合并运算符 ?? 提供默认值：value.year ?? 0。

编译错误 'id' does not exist in BillItem props / 'size' does not exist in PieChart props
✅ 解决方案

组件属性名不匹配：BillItem 的 id 应为 billId，PieChart 的 size 应为 chartSize，LineChart 的 width 应为 canvasWidth，BarChart 的 width 应为 chartWidth。

功能问题 TabBar 切换页面后返回键无效
✅ 解决方案

TabBar 使用 router.replaceUrl() 会替换当前页面导致导航栈为空，改为 router.pushUrl() 保留页面栈，使 router.back() 能正确返回上一页。

编译错误 Property 'flexWrap' does not exist on type 'RowAttribute'
✅ 解决方案

ArkTS 的 Row 组件不支持 flexWrap 属性，改用 Flex({ wrap: FlexWrap.Wrap }) 组件实现换行布局。

编译错误 Unexpected token — @Builder 方法中 ForEach 回调导致解析失败
✅ 解决方案

ArkTS 的 @Builder 方法中 ForEach 使用 lambda 回调可能触发解析器异常。将 ForEach 替换为显式的多个组件实例，或将逻辑提取到独立方法中。同时将弹窗浮层从 build() 根节点外移入 Stack 内。
